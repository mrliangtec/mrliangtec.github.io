假期微信小程序实训笔记

Day1：Git本地环境的搭建，用Git管理项目

1. 提交代码的正确顺序

  1. 使用git add 文件名.文件后缀将文件添加到本地Git缓存区
  2. 使用git status查看状态
  3. 使用git commit -m '提交说明'将修改推送到本地Git库中
  4. 使用git remote add origin 远程仓库url将本地Git仓库与代码托管平台上的仓库连接
  5. 使用git push -u origin master将本地Git仓库代码推送到代码托管平台（Github，Gitee，Coding等）

或者直接使用Github Desktop

2. 使用Git将代码提交到Github（完整代码示例）

说明：

  1. 最后一行中的-u origin master只有第一次提交时才需要添加
  2. 首次执行git push -u origin master时，系统会弹出对话框，这时需要登陆自己的Github账户
  3. 虽然创建项目时Github给出的代码为git push -u origin main，但在实际操作中git push -u origin main会报错，所以需要将main修改为master
  4. 提交过程中可能会报错：OpenSSL SSL_read: Connection was reset, errno 10054，解决代码如下：
    git config --global http.sslVerify "false"

3. “git clone”

使用方法：git clone 远程仓库url，回车
执行clone之后，执行的目录会出现一个以仓库名为文件夹名的文件夹，里面是clone的远程仓库的文件，以及仓库的连接，不需要上述手动创建，方便快捷。
使用时，为了少给自己挖坑，为了避免冲突，使用git的时候，一定养成一个好习惯，先pull，再push。

4. 作业

  1. 注册Github账号，并把注册邮箱发给同时@老师。（状态：已完成。）
  2. 下载安装Visual Studio Code编辑器 vscode官网 。（状态：已完成。）

Day2：CSS（复习?）

前端技术

前端三大核心技术：Html，CSS，JavaScript（EcmaScript，DOM，BOM）
JavaScript：全球唯一一门真正意义上的全栈开发语言（Java是后端语言）
Html负责网页的结构，CSS负责网页的美化，JavaScript负责网页的特效。

CSS简介

CSS：层叠样式表（层叠：样式可以覆盖），英文全称Cascading Style Sheets
CSS作用：用来定义如何来显示Html元素
通常样式是存储在样式表中（外部CSS文件引入），CSS文件后缀：.css

CSS三种引入方式

  • 行间CSS（内联样式）
  • 页面级CSS（内部样式）
  • 外部CSS文件引入（外部样式）

优先级：行间样式 > 内部样式 > 外部样式

CSS基本语法

CSS语法由两个部分组成：选择器，一条或多条声明。
选择器：告诉CSS找到元素的方式。
声明：写在{}中。每个声明由一个属性和一个值组成，二者之间用:隔开，多个声明是由;分开。
